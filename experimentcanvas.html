HTML2CANVAS TESTING

html2canvas (https://html2canvas.hertzen.com/) is a free plugin that simulates creating a screenshot from the designated HTML source. IT IS NOT A REAL SCREENSHOT. It's basically just reading off the HTML/CSS lines recorded on the document and regenerating it into a new "canvas" element. (https://www.w3schools.com/html/html5_canvas.asp) When canvas is generated, either user can save it manually by right clicking and stuff... or we (HiNeon) store it on our server using more script. Otherwise, canvas is NOT stored anywhere and is gone forever with the page refresh.

Trouble implementing with our web app at the moment

The background image slider, slides through images/divs by changing its position - using css: translate3D property. This property is unfortunately not supported by html2canvs plugin at the moment... Hence it's showing the bare background inside the slider viewport div. I can make the viewport background: black so that the letters can stand out better on screenshot but that's about it. We either change the slider plugin or html2canvas plug in, one of the other has to be gone.
